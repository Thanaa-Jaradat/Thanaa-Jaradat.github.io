Projects

Go Team

Implemented using: Java, Spring Boot, Spring MVC, Spring Security, REST client, React.Js, CSS
A full front-end back-end match scheduling and management website where users can create an account and sign-in to schedule or join a match. Users can select from a list of sports and schedule a new match based on user-specific conditions and available times, users also can see all the available matches or filter them based on their conditions, they can also select a game and join it. All the games, users and times are stored and managed in an SQL database.

Jac Enterprises

Implemented using: Java, JavaScript, React.JS, HTML, CSS
A React.Js front-end that allows the users to create and submit requests using buttons and dropdown lists to three different APIs, Cryptid API,The Metropolitan Museum of Art, and Bored API. These APIs return responses containing Objects like images or texts. The front-end parses and displays the returned object. The Cryptid API is a custom made API that receives requests for a cryptid and responds back with either a random or a user specified cryptid.

Created a web application, inspired by “Cookie Clicker,” which allows users to track the number of donut clicks and purchase multipliers and autoclickers.
Designed in HTML & styled with CSS, uses JavaScript to track information about total clicks, multiplier effects, number of autoclickers.

Virtual Pet

Implemented using: Java, Spring boot, OOP, HTML, CSS, JavaScript
A java program that creates a virtual pet where the player can choose between having a cat or a dog, a robotic pet or an organic one. The player of this game needs to maintain the status of their pet by feeding, and taking care of them. For each type of pet, the status depends on properties such as: Hunger, thirst, boredom and hygiene levels for the organic pets and oil and maintenance levels for the robotic ones.

Movie Review API

Implemented using: Java, Spring Boot, Spring MVC, REST client
Movie review API that receives a request holding a user review on a certain movie, the review in the request is composed of textual review, numerical rating, review author name and other attributes, the API extracts all of these attributes from the request and applies the requested change (e.g insert, update, select… etc) on the data stored in an SQL database using Hibernate query language, then a response is sent back to the client containing a confirmation of the change in the request.
